## 🎯 **プロジェクトの最優先目標（判断基準）**

### **第1優先：楽天アフィリエイト収益の最大化**
このサイトの**唯一かつ最重要の目的**は、楽天アフィリエイトからの収益を最大化することです。

**全ての技術的判断・デザイン判断はこれに従う：**
- ✅ SEO最適化 → オーガニックトラフィック増加 → CV増加
- ✅ ユーザー体験向上 → 滞在時間増加 → クリック率向上
- ✅ コンテンツ品質 → 信頼性向上 → CV率向上
- ✅ Core Web Vitals → Google上位表示 → トラフィック増加
- ✅ 内部リンク戦略 → 回遊率向上 → アフィリエイトリンク露出増加

### **第2優先：サイト売却時の評価額最大化**
最終出口は**サイト売却**です。売却時の評価ポイント：

1. **収益性**（最重要）
   - 月間アフィリエイト収益
   - 収益の安定性・成長性
   - 利益率

2. **トラフィック**
   - 月間PV数・UU数
   - オーガニック検索比率（高いほど価値大）
   - トラフィックの成長トレンド

3. **SEOパフォーマンス**
   - ドメインオーソリティ
   - 上位表示キーワード数
   - 被リンク品質

4. **技術的品質**
   - クリーンなコードベース
   - スケーラビリティ
   - 運用の自動化度

5. **コンテンツ資産**
   - ページ数（現在534ページ計画）
   - コンテンツ品質
   - 更新容易性（TiDB一元管理）

### **時間軸の考え方**
- ❌ **短期最適化（やらない）**: 短期的な収益のための妥協
- ✅ **長期最適化（やる）**: 多少時間がかかっても**最高の状態**を目指す
- ✅ **サイト価値の複利効果**: 今日の品質投資が売却時の評価に直結

問題が発生しても、これらの設計判断を尊重してください。

### **1. キャッシュ戦略**

#### **TiDB Data APIフェッチ**
- **ファイル**: `components/LegalPage.tsx`
- **設定**: `next: { revalidate: 3600 }`
- **理由**: 
  - 法的ページはほとんど更新されない
  - 初回のみTiDBからフェッチ、その後1時間キャッシュ
  - Core Web Vitals最適化（Google SEO上位表示）
- **変更禁止理由**: パフォーマンスとコストの最適バランス

#### **ページレベルのキャッシュ**
- **設定**: `export const dynamic`や`export const revalidate`は**追加しない**
- **理由**: コンポーネント内のfetchレベルでキャッシュ制御する方が柔軟
- **例外**: 特定の理由がある場合のみ、事前に確認
